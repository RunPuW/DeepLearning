## Schema

Each record has 9 fields after splitting (8 original + `hard_subset`):

| Field | Type | Description |
|---|---|---|
| `uid` | string | Globally unique identifier, e.g. `fpb_00001` |
| `dataset` | string | Source dataset: `fpb`, `sentfin`, `finentity`, `finmarba` |
| `task_type` | string | `sentence_semantic`, `target_semantic`, or `market_aux` |
| `text` | string | Financial news sentence or headline |
| `label` | string\|null | Sentiment: `positive`, `neutral`, `negative`. Null for `market_aux` |
| `target` | string\|null | Named entity for target tasks. Null for `sentence_semantic` and `market_aux` |
| `market_label` | string\|null | Aggregate market sentiment for `market_aux` only |
| `split` | string | `train`, `val`, `test`, or `analysis` |
| `hard_subset` | string | Pipe-separated subset tags or empty string. See below |

## Hard Subset Definitions

The `hard_subset` field tags rows belonging to challenging evaluation subsets:

### `multi_entity`
- **Applies to:** `target_semantic` rows
- **Definition:** The text contains â‰Ą2 distinct named entities as targets in the corpus
- **Challenge:** Models must correctly attribute sentiment to each entity independently, not conflate

### `conflict`
- **Applies to:** `target_semantic` rows
- **Definition:** The same `(text, target)` pair appears with â‰Ą2 different labels across the corpus
- **Challenge:** Reflects genuine annotator disagreement or polysemy; tests model calibration under label noise

### `ambiguous`
- **Applies to:** `sentence_semantic` rows
- **Definition:** The same text appears with â‰Ą2 different labels in the corpus (FPB boundary agreement cases)
- **Challenge:** Intrinsically ambiguous financial language; tests model uncertainty handling

## Bias Checklist

The following checks are performed automatically by `make_splits.py` and recorded in `split_spec.json`:

| Check | Method | Expected |
|---|---|---|
| **Exact duplicate leakage** | Exact text match between val/test and train | 0 matches |
| **Near-duplicate leakage** | 8-gram Jaccard â‰Ą 0.85, sampled | Minimal |
| **UID uniqueness** | Each UID appears in exactly one split | No duplicates |
| **Label balance** | % positive/neutral/negative per split | Stratified ~equal |
| **Future information leakage** | Timestamps in source data | N/A â€” no timestamps |
| **Split pollution** | No sample in multiple splits | OK |

> See `split_spec.json â†’ bias_checklist` for actual computed values.

## Known Limitations

1. **Temporal concentration:** FPB texts are predominantly from Finnish/Nordic companies, 2004â€“2011. Domain shift risk for other markets.
2. **Single-sentence focus:** All texts are sentence-level; paragraph or document-level inference is out of scope.
3. **Annotator agreement threshold:** FPB uses the 50% agreement threshold (not 75% or 100%). Approximately 15â€“20% of FPB instances are genuinely ambiguous.
4. **FinMarBa exclusion:** Market-level labels are auxiliary and not validated for per-sentence accuracy. Do not use for supervised training without additional filtering.
5. **Entity boundary variation:** SEntFiN and FinEntity use different entity mention styles; target strings are not normalized.
