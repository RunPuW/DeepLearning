## Proposed Changes

### Split & Freeze Script

#### [NEW] make_splits.py (file:///c:/Users/Administrator/Desktop/整合+预处理数据(1)/整合+预处理数据/make_splits.py)

Performs:
1. **Seed-locked** stratified splitting on semantic rows only (`task_type != market_aux`)
2. Split ratios: **70 / 15 / 15** (train / val / test) — stratified by [(dataset, label)](file:///c:/Users/Administrator/Desktop/%E6%95%B4%E5%90%88+%E9%A2%84%E5%A4%84%E7%90%86%E6%95%B0%E6%8D%AE%281%29/%E6%95%B4%E5%90%88+%E9%A2%84%E5%A4%84%E7%90%86%E6%95%B0%E6%8D%AE/eda.py#480-500) to preserve source and class balance
3. `market_aux` rows → assigned `split=analysis` (unchanged, excluded from eval)
4. **Hard subset tagging** (`hard_subset` field):
   - `multi_entity`: `target_semantic` rows where the same [text](file:///c:/Users/Administrator/Desktop/%E6%95%B4%E5%90%88+%E9%A2%84%E5%A4%84%E7%90%86%E6%95%B0%E6%8D%AE%281%29/%E6%95%B4%E5%90%88+%E9%A2%84%E5%A4%84%E7%90%86%E6%95%B0%E6%8D%AE/preprocess.py#64-73) has ≥2 distinct targets
   - `conflict`: [(text, target)](file:///c:/Users/Administrator/Desktop/%E6%95%B4%E5%90%88+%E9%A2%84%E5%A4%84%E7%90%86%E6%95%B0%E6%8D%AE%281%29/%E6%95%B4%E5%90%88+%E9%A2%84%E5%A4%84%E7%90%86%E6%95%B0%E6%8D%AE/eda.py#480-500) pairs that appear with ≥2 different labels across the corpus
   - `ambiguous`: `sentence_semantic` rows where annotator agreement is known to be boundary (FPB 50% agree threshold rows are all in dataset, but sentences that appear with both pos/neg in the corpus get flagged)
5. **Bias checklist** computed and embedded in `split_spec.json`:
   - Exact-duplicate cross-split leakage (text match across train/val/test)
   - Near-duplicate detection via 8-gram fingerprint (MinHash sketch, threshold 0.85)
   - Future-info leakage: N/A (no timestamps in FPB/SEntFiN/FinEntity; FinMarBa market aux excluded from eval)
   - Split pollution: verify no UID appears in multiple splits
   - Label balance per split
6. Writes `split_spec.json` with full provenance, checksums (MD5 per split file), and bias checklist results

Documents:
- Primary metric: **macro-F1** on test split (semantic tasks only)
- Secondary metrics: per-class F1, accuracy, MCC, per-dataset macro-F1
- Hard subset metrics: separate macro-F1 on multi_entity / conflict / ambiguous subsets
- Bias checklist results (referenced from split_spec.json)
- Reporting requirements: must report all metrics to be considered valid evaluation

### Evaluation Script

#### [NEW] run_eval.py (file:///c:/Users/Administrator/Desktop/整合+预处理数据(1)/整合+预处理数据/run_eval.py)

CLI script:
```
python run_eval.py \
  --predictions predictions.jsonl \
  --split_file split_spec.json \
  --data_file final_integrated.jsonl \
  --split test \
  --output_dir eval_results/
```

Outputs:
- `eval_results/metrics.json` — all numeric metrics
- `eval_results/metrics_by_dataset.csv` — per-dataset breakdown
- `eval_results/confusion_matrix.csv` — 3×3 confusion matrix
- `eval_results/hard_subset_metrics.json` — hard subset breakdown
- `eval_results/eval_report.md` — human-readable summary
